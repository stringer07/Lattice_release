# Task completion checklist
- Confirm edited website files live under `docs/` so GitHub Pages can publish them.
- Verify newly referenced assets exist inside `docs/assets/`.
- Run `rtk node --check docs/site.js` after JS edits.
- Review links between `docs/index.html` and `docs/privacy.html`.
- Check `git status --short docs` before finishing so new static assets are not missed.
- If discussing GitHub Pages capabilities, prefer current official GitHub Docs rather than memory.
